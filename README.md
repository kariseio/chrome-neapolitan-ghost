# 탭을 옮겨다니는 것 (PoC)

나폴리탄 괴담식 크롬 확장 프로토타입. 점프스케어 없음.
**설치하는 순간** 열어둔 탭 중 하나에 "귀신"이 깃들고, 동시에 **규칙서**가 열린다.
규칙을 어길수록 그것은 **더 고장난다.**

## 흐름
1. 설치 → 규칙서 탭이 뜨고, http/https 탭 하나의 favicon이 눈으로 바뀐다.
2. 귀신은 한 번에 한 탭(초반)에만 깃들고, 수십 초마다 다른 탭으로 옮겨간다.
3. 규칙을 어기면 **오염도(corruption 0→8)**가 오른다. 가만히 둬도 이동할 때마다 **저절로 조금씩** 오른다(드리프트).

## 오염 단계 (0~8, 오를수록 증상이 하나씩 얹힌다)
| 단계 | 상태 | 새로 얹히는 증상 |
|---|---|---|
| 0 | 온전함 | 눈 favicon, 느린 깜빡임, 아주 드문 속삭임 |
| 1 | 실금 | 속삭임·깜빡임 증가, 결번 04의 정체가 드러남 |
| 2 | 벌어짐 | **비네트(가장자리 어둠)**, 제목 글리치, 눈 2개, 규칙 07 등장(03과 대립) |
| 3 | 새어 나옴 | **화면 밝기 깜빡임 시작**(형광등 죽어가듯), 규칙 11·14 등장 |
| 4 | 스며듦 | 깜빡임 잦아짐, 눈 3개, 시야가 조여옴 |
| 5 | 번짐 | **암전(lights-out)**, 화면이 **회색으로** 물들기 시작, 규칙 16 등장 |
| 6 | 일그러짐 | **화면 미세 떨림**, 색이 틀어짐(hue), 눈 4개 |
| 7 | 무너지는 중 | 잦은 암전, 시야가 터널처럼 좁아짐, 제목 붕괴 |
| 8 | 무너짐 | 거의 상시 깜빡임·떨림·짙은 회색, "규칙은 없다" |

규칙서 문서 자체도 오염도에 따라 어두워지고 깜빡이며, 문구가 훼손되고 **없던 규칙**이 스며 나온다.

## 규칙과 대립
규칙 번호는 **고정**이고 4·12·15는 결번(缺番)이다. 규칙끼리 서로 어긋난다.

**실제로 처벌되는 규칙**(어기면 오염도 +1, 규칙서에 취소선):
| 규칙 | 내용 | 감지 방식 |
|---|------|-----------|
| 02 | 규칙서 탭을 닫지 말 것 | `tabs.onRemoved` |
| 03 | 눈 탭을 직접 닫지 말 것 | `tabs.onRemoved` |
| 05 | 눈 탭을 30초 이상 응시하지 말 것 | content 가시성 타이머 |
| 08 | 탭을 7개 이상 열지 말 것 | `tabs.onCreated` |

**주요 대립** (규칙서에는 **절대 라벨로 표시하지 않는다** — 독자가 스스로 눈치채야 공포가 산다):
- **03 ↔ 07**: 닫지 마 ↔ 눈이 둘 이상이면 하나는 반드시 닫아라 → 오염도 2부터 뭘 해도 위반(죽음의 나선).
- **05 ↔ 06/14**: 시선 돌려라 ↔ 시선 떼지 마라 (처벌은 05 쪽만 → 어느 쪽이 진짜인지 모름).
- **02 ↔ 09**: 문서 닫지 마 ↔ 다 닫아라.
- 나머지(06·09·10·11·16 등)는 분위기용(감지 불가라 더 불공평하게 느껴짐).
- 메타 규칙 13은 "규칙이 어긋나거든 더 낮은 번호를 따르라"고만 할 뿐, *어디가* 어긋나는지는 말하지 않는다.

오염도는 `storage.local`에 저장되어 **브라우저를 껐다 켜도 유지된다**(고장 누적).

## 설치 (개발자 모드)
1. 주소창 → `chrome://extensions`
2. 우상단 **개발자 모드** ON
3. **압축해제된 확장 프로그램을 로드** → 이 폴더(`D:\study\chrome-ghost`) 선택
4. 규칙서가 뜬다. http/https 탭 여러 개 열고 관찰.

## 테스트 팁
- **기존에 열려 있던 탭**: content script는 확장 로드 "이후" 새 탐색/새로고침부터 자동 주입된다. 하지만 귀신이 그런 탭을 방문하면 `scripting`으로 **자동 주입 후 재시도**하므로 결국 영향을 받는다. 바로 보고 싶으면 대상 탭을 F5.
- **높은 단계 빨리 보기**: 규칙을 일부러 여러 번 어겨라(탭 8개 열기, 눈 탭 닫기 반복). 오염도가 오르며 규칙서가 실시간으로 변한다.
- **드리프트**: 가만히 둬도 이동할 때마다 `DRIFT_CHANCE`(기본 15%) 확률로 저절로 오른다. 천천히 나빠지므로, 급하면 규칙을 어겨서 올려라.
- **속도 조절**: `background.js`의 `DRIFT_CHANCE`, `params()` 지연값, `content.js`의 `cfg()` 곡선으로 강도/속도를 조절.
- **초기화**: 오염도는 누적·지속되므로, 규칙서 우하단의 희미한 `·` 버튼(개발용)으로 0으로 리셋.
- **규칙서 다시 열기**: `chrome://extensions`에서 확장 ID 확인 후 `chrome-extension://<ID>/rules.html`.

## 끄기
`chrome://extensions`에서 토글 OFF 또는 삭제. content script가 흔적(favicon·오버레이)을 복원한다.
오염도까지 지우려면 규칙서의 리셋 버튼을 쓰거나 확장을 삭제하라.

## 파일
- `manifest.json` — 권한/구성 (Manifest V3)
- `background.js` — 서비스 워커. 귀신 위치·오염도 관리, 규칙 위반 감지.
- `content.js` — 페이지 주입. 강도(intensity)별 연출.
- `rules.html` / `rules.js` — 규칙서. 오염도에 따라 훼손된다.

## 알려진 한계 (PoC)
- SW는 상주하지 않음 → `chrome.alarms`로 깨어나므로 이동 간격이 정확히 일정하진 않다.
- favicon이 없는 사이트는 귀신이 떠난 뒤 아이콘이 잠시 비어 보일 수 있다.
- 속삭임/오버레이는 일부 프레임워크(React 등) 페이지에서 즉시 덮어써질 수 있다(해는 없음).
- 규칙 위반 판정은 "사용자가 닫았다"고 가정한다(확장은 스스로 탭을 닫지 않음).
